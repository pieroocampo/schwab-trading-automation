# Schwab Trading Automation Setup Guide

## Required Packages

### Core dependencies

Install everything from the repo root:

```bash
pip install -r requirements.txt
```

That pulls in **schwab-py**, **databricks-sdk**, **python-dotenv**, and optional Databricks-related packages listed in `requirements.txt`. You only need the Databricks pieces if you use `order_export.py` upload/job features.

### Installation

1. Create a virtual environment (recommended):

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate
   ```

2. From the repository root:

   ```bash
   pip install -r requirements.txt
   ```

## Environment configuration

Scripts load `.env` from the **current working directory** via `python-dotenv`. The project expects a `.env` under **`schwab-proj/`** (see repository `.gitignore`). Always run tools from that directory unless you copy or symlink `.env` elsewhere.

### Schwab (required for any script that talks to the API)

```bash
SCHWAB_CLIENT_ID=your_client_id_here
SCHWAB_CLIENT_SECRET=your_client_secret_here
CALLBACK_URL=https://127.0.0.1:8182/callback
```

Optional: `TOKEN_PATH` (default `token.json`, relative to cwd).

### Order export (`order_export.py`)

```bash
OUTPUT_FILE=filled_orders.csv
CUTOFF_DATE=2024-06-15T00:00:00+00:00

DATABRICKS_PROFILE=mypharos
DATABRICKS_REMOTE_PATH=/Volumes/workspace/default/landing/filled_orders.csv
DATABRICKS_JOB_ID=939121727711316
```

### Trading automation (`order_manager.py`)

`TICKERS` is **required** (comma-separated symbols, e.g. `AAPL,MSFT`).

Optional: `DRY_RUN=true`, `DEBUG=true`, technical and risk parameters (see `schwab-proj/config.py` and `TradingConfig`). Peak giveback: `GIVEBACK_PCT`, `GIVEBACK_ACTIVATION_PCT`, `PEAK_STATE_PATH` (documented in the main readme).

### Logging

Optional: `LOG_LEVEL`, `LOG_FILE_ENABLED`, `LOG_CONSOLE_ENABLED`, etc. (see `load_logging_config` in `schwab-proj/config.py`).

### Databricks CLI (only if using uploads / job trigger)

```bash
databricks auth login --host https://your-databricks-workspace.cloud.databricks.com
databricks auth profiles
```

## Verification

From `schwab-proj`:

```bash
cd schwab-proj
python3 -c "from config import validate_environment; print('OK' if validate_environment() else 'Missing env')"
```

`validate_environment()` only checks Schwab API variables; `order_manager.py` still needs `TICKERS` set.

## Usage

```bash
cd schwab-proj
python3 order_export.py
python3 order_manager.py
python3 transform_history.py
```

## Troubleshooting

1. **Import errors:** Activate the venv and `pip install -r requirements.txt` from the repo root.
2. **Schwab auth:** Confirm `.env` in `schwab-proj`, callback URL matches your Schwab app, and delete `token.json` there to force a new OAuth flow.
3. **Databricks:** Confirm CLI profile matches `DATABRICKS_PROFILE` and paths/job id are valid for your workspace.
4. **Logs:** `schwab-proj/order_export.log`, `schwab-proj/trading.log` (paths follow cwd when file logging is enabled).
